# Onboarding Templates

**Purpose**: Reusable templates and checklists for consistent onboarding
**Scope**: All roles, scenarios, and validation processes
**Integration**: Links with project-wide @99_templates/

## Template Categories

### Role-Specific Templates
Ready-to-use instruction templates for each agent role
- Orchestrator agent instruction template
- Manager agent instruction template  
- Specialist agent instruction template
- Environment master certification template

### Onboarding Checklists
Structured validation for learning progress
- Foundation module completion checklist
- Role track validation checklist
- Environment mastery assessment checklist
- Escalation protocol familiarity checklist

### Best Practice References
Quick reference materials for common scenarios
- Context preservation best practices
- Quality gate validation procedures
- Technical implementation standards
- Communication and reporting templates

## Template Structure

Each template includes:
- **Purpose and Scope**: Clear definition of when to use
- **Prerequisites**: Required knowledge or completed modules
- **Step-by-Step Instructions**: Detailed procedures to follow
- **Validation Criteria**: How to confirm successful completion
- **Common Pitfalls**: Known issues and how to avoid them
- **Integration Points**: Links to related materials and processes

## Usage Guidelines

### For New Agents
1. Start with foundation module templates
2. Progress through role-specific materials
3. Use checklists to validate understanding
4. Reference best practices during initial work
5. Follow escalation templates when needed

### For Experienced Agents
1. Use templates for role transitions
2. Reference during complex scenarios
3. Contribute improvements based on experience
4. Mentor others using template guidance
5. Help maintain template currency and accuracy

### For Managers and Orchestrators
1. Use templates to assess agent readiness
2. Customize templates for specific scenarios
3. Track completion and identify gaps
4. Update templates based on lessons learned
5. Ensure templates align with project needs
